# 热敏单量统计业务核心分析

## 1. 核心数据表及关系

### 1.1 主要数据表

#### `cfg_logistics_num` - 热敏物流单剩余单量表（主表）
**表结构：**
- `rec_id` - 主键，自增ID
- `deleted` - 是否删除（0:未删除, 1:已删除）
- `logistics_id` - 物流公司ID（关联 `cfg_logistics.rec_id`）
- `logistics_no` - 物流编号
- `logistics_name` - 物流名称
- `logistics_type` - 物流类型ID（关联 `dict_logistics.rec_id`）
- `bill_type` - 单号类型（0:普通, 1:线下电子面单, 2:云栈热敏, 3:京东阿尔法, 4:拼多多, 5:唯品会, 6:区域零售, 7:导入热敏单号, 8:云集电子面单, 9:菜鸟ASCP单号, 10:抖音电子面单号, 11:TMS单号, 12:快手电子面单号, 13:得物直发, 14:唯品会, 15:华为商城, 16:小红书, 17:微信视频号, 18:爱库存电子面单）
- `quantity` - **剩余单量**（核心字段）
- `alarm_quantity` - **预警值**（核心字段）
- `branch_code` - 网点编号
- `branch_name` - 网点名称

**唯一索引：** `UK_cfg_logistics_no_and_brand_code` (`logistics_no`, `branch_code`)

#### `cfg_logistics_num_detail` - 热敏物流单剩余单量网点详细地址表（关联表）
**表结构：**
- `rec_id` - 主键，自增ID
- `logistics_branch_id` - 关联 `cfg_logistics_num.rec_id`
- `province` - 省份
- `city` - 城市
- `area` - 区域
- `town` - 街道/镇
- `address_detail` - 详细地址

**索引：** `IX_cfg_logistics_num_detail_branch_id` (`logistics_branch_id`)

#### `cfg_logistics_num_log` - 热敏单量统计日志表（日志表）
**表结构：**
- `rec_id` - 主键，自增ID
- `logistics_num_relation_id` - 关联 `cfg_logistics_num.rec_id` 或预警提醒员工表 rec_id
- `operator_id` - 操作员ID（关联 `cfg_employee.rec_id`）
- `type` - 操作类型（1:维护预警值日志, 2:维护预警员工日志）
- `message` - 日志消息
- `created` - 创建时间

**索引：** `UK_cfg_logistics_num_log_relation_id` (`type`, `logistics_num_relation_id`)

### 1.2 关联表

#### `cfg_logistics` - 物流公司表
- 通过 `cfg_logistics_num.logistics_id` 关联
- 存储物流公司的基本信息

#### `dict_logistics` - 物流类型字典表
- 通过 `cfg_logistics_num.logistics_type` 关联
- 存储物流类型信息

### 1.3 表关系图

```
cfg_logistics (物流公司)
    ↑
    | logistics_id
    |
cfg_logistics_num (热敏物流单剩余单量) ←──┐
    |                                    │

    |                                    │
    ├──→ cfg_logistics_num_detail (网点详细地址)

dict_logistics (物流类型)
    ↑
    | logistics_type

```

## 2. 关键业务流程

### 2.1 查询热敏单量统计流程

**入口：** `StockThermalStatWindow` (热敏单量统计窗口)

**流程：**
1. **查询条件：**
   - 物流名称（模糊查询）
   - 物流编号（精确查询）
   - 物流类型（下拉选择）
   - 网点编号（模糊查询）
   - 剩余单量低于预警值（复选框筛选）

2. **查询存储过程：** `SP_CFG_LOGISTICS_NUM_QUERY`
   - 根据查询条件动态构建SQL
   - 支持分页查询
   - 返回字段：`rec_id`, `logistics_id`, `logistics_no`, `logistics_name`, `branch_code`, `branch_name`, `logistics_type`, `bill_type`, `alarm_quantity`, `quantity`

3. **数据展示：**
   - 主列表：显示物流网点及剩余单量信息
   - 网点地址标签页：显示选中网点的详细地址信息（通过 `cfg_logistics_num_detail_query_by_branch_id` 查询）
   - 日志标签页：显示操作日志（通过 `cfg_logistics_num_log_query` 查询）

### 2.2 维护预警值流程

**入口：** `StockThermalSetAlarmQuantityWindow` (维护预警值窗口)

**流程：**
1. **选择网点：** 在热敏单量统计列表中选择需要维护预警值的物流网点
2. **打开维护窗口：** 点击"维护预警值"按钮，打开维护窗口
3. **编辑预警值：** 在表格中直接编辑 `alarm_quantity` 字段
4. **保存操作：** 
   - 调用存储过程 `cfg_logistics_num_update`
   - 更新 `cfg_logistics_num.alarm_quantity` 字段
   - 记录操作日志到 `cfg_logistics_num_log`（type=1）

**存储过程逻辑：**
```sql
UPDATE cfg_logistics_num cln 
INNER JOIN tmp_xchg tx ON cln.rec_id = tx.f1 
SET cln.alarm_quantity = tx.f2;

-- 记录日志
INSERT INTO cfg_logistics_num_log(logistics_num_relation_id, operator_id, type, message)
SELECT tx.f1, @cur_uid, 1, CONCAT(cln.branch_name, "网点维护预警值 ", tx.f2)
FROM tmp_xchg tx
INNER JOIN cfg_logistics_num cln ON cln.rec_id = tx.f1
WHERE tx.f3 = 1;
```

### 2.3 获取热敏单号流程（减少单量）

**入口：** 多个业务场景（装箱、打印、验货等）

**流程：**
1. **调用接口：** 通过 `Logistics.GetPackOrderLogisticsNo()` 方法获取物流单号
2. **接口调用：** `sys_ext_cmd` → `logistics` → `pack_get_waybill` 或 `multi_get_waybill`
3. **减少单量：** 获取热敏单号成功后，外部接口或存储过程会减少对应网点的 `cfg_logistics_num.quantity` 字段
4. **更新状态：** 将获取到的物流单号更新到装箱单或出库单

**关键代码位置：**
- `WdtWMS\csharp\WdtWMS.Control\Common\Business\Logistics\Logistics.cs`
- 多个业务窗口调用：`StockSplitOrderWindow`, `StockSalesPrintWindow`, `StockoutSalesOrderWindow` 等

### 2.4 数据同步流程

**说明：** 热敏单量数据通常通过外部接口同步更新
- 从物流平台接口获取剩余单量
- 更新 `cfg_logistics_num.quantity` 字段
- 同步网点地址信息到 `cfg_logistics_num_detail`

## 3. 数据修改的业务

### 3.1 维护预警值（手动修改）

**业务操作：** 在热敏单量统计窗口中维护预警值

**修改的表：**
- `cfg_logistics_num.alarm_quantity` - 预警值字段

**影响的业务：**
- 预警提醒：当 `quantity < alarm_quantity` 时，系统会触发预警提醒
- 查询筛选：支持按"剩余单量低于预警值"条件筛选

**相关代码：**
- `StockThermalStatWindow.OnClickSetAlarmQuantity()` - 打开维护窗口
- `StockThermalSetAlarmQuantityWindow.OnClickSave()` - 保存预警值
- 存储过程：`cfg_logistics_num_update`

### 3.2 获取热敏单号（自动减少单量）

**业务操作：** 在装箱、打印、验货等环节获取热敏单号

**修改的表：**
- `cfg_logistics_num.quantity` - 剩余单量字段（减少）

**触发场景：**
1. **装箱时自动获取：** 装箱完成后自动获取物流单号
2. **批量获取物流单号：** 在打印窗口批量获取物流单号
3. **验货时获取：** 验货环节获取物流单号
4. **手动获取：** 用户手动点击"获取物流单号"按钮

**关键方法：**
- `Logistics.GetPackOrderLogisticsNo()` - 获取单个装箱单物流单号
- `Logistics.GetPackOrderLogisticsNo(List<long> packIds, ...)` - 批量获取物流单号

**调用位置：**
- `StockSplitOrderFasterCreateWindow` - 快速创建拆分单
- `StockSalesPrintWindow` - 打印窗口
- `StockoutSalesOrderWindow` - 销售出库单窗口
- `StockSplitOrderWindow` - 拆分单窗口
- 等多个业务窗口

### 3.3 同步热敏单量（外部接口同步）

**业务操作：** 从物流平台接口同步剩余单量数据

**修改的表：**
- `cfg_logistics_num.quantity` - 剩余单量字段（更新）
- `cfg_logistics_num_detail` - 网点地址信息（可能新增/更新）

**同步方式：**
- 通过外部接口调用获取最新单量
- 更新数据库中的剩余单量

### 3.4 网点地址维护

**业务操作：** 维护网点的详细地址信息

**修改的表：**
- `cfg_logistics_num_detail` - 网点详细地址表

**操作说明：**
- 一个网点可以对应多个地址（省、市、区、街道/镇、详细地址）
- 通过 `cfg_logistics_num_detail_query_by_branch_id` 查询网点地址列表

## 4. 关键业务规则

### 4.1 预警机制
- 当 `quantity < alarm_quantity` 时，系统会触发预警
- 支持在查询时筛选"剩余单量低于预警值"的记录
- 预警消息可以通过消息提醒功能通知相关人员

### 4.2 单量扣减规则
- 每次成功获取热敏单号，对应网点的剩余单量会减少
- 单量扣减在获取单号成功后执行
- 如果获取单号失败，不会扣减单量

### 4.3 数据唯一性
- 通过 `(logistics_no, branch_code)` 唯一索引保证同一物流编号和网点编号的唯一性
- 一个物流公司可以有多个网点，每个网点有独立的剩余单量

### 4.4 日志记录
- 维护预警值操作会记录日志（type=1）
- 日志包含操作人、操作时间、操作内容等信息
- 支持查询历史操作记录

## 5. 相关权限

- `stock_eorder_stat` - 热敏单量统计（主权限）
- `stock_eorder_stat_search` - 搜索
- `stock_eorder_stat_set_alarm_quantity` - 维护预警值
- `stock_eorder_stat_set_message_employee` - 预警消息提醒

## 6. 总结

热敏单量统计业务的核心是管理热敏物流单的剩余单量，主要涉及：
1. **数据存储：** 通过 `cfg_logistics_num` 表存储剩余单量，通过 `cfg_logistics_num_detail` 存储网点地址
2. **数据查询：** 支持多条件查询和预警筛选
3. **数据修改：** 主要通过获取热敏单号时自动减少单量，以及手动维护预警值
4. **日志记录：** 所有关键操作都会记录日志，便于追溯

该业务与出库、装箱、打印等业务流程紧密关联，是物流单号管理的重要组成部分。
