# 销售订单业务核心分析

## 1. 核心数据表及关系

### 1.1 主要数据表

#### **sales_trade（销售订单主表）**
- **主键**: `trade_id` (bigint)
- **唯一键**: `UK_trades_trade_no` (`trade_no`, `owner_id`, `split_no`, `origin_owner_id`)
- **核心字段**:
  - `trade_no`: OMS的订单编号
  - `trade_sys_no`: 系统订单编号
  - `owner_id`: 货主
  - `warehouse_id`: 仓库
  - `trade_status`: 订单状态（5已取消 10等递交 30待客审 55已确认 95已发货）
  - `source_id`: 订单来源（0手工导入 大于0接口下发）
  - `platform_id`: 平台ID
  - `shop_id`: 店铺ID
  - `logistics_id`: 物流公司ID
  - `buyer_nick`: 买家网名
  - `receiver_*`: 收件人信息（姓名、地址、手机等）
  - `total_amount`: 订单总金额
  - `goods_amount`: 货款总额
  - `post_amount`: 邮费
  - `discount`: 优惠金额
  - `paid`: 已付金额
  - `freeze_reason`: 冻结原因
  - `cancel_reason`: 取消原因

#### **sales_trade_order（销售订单明细表/子订单表）**
- **主键**: `rec_id` (bigint)
- **外键**: `trade_id` → `sales_trade.trade_id`
- **唯一键**: `UK_sales_trade_order` (`trade_id`, `spec_id`, `oms_oid`)
- **核心字段**:
  - `trade_id`: 关联订单主表
  - `spec_id`: 规格ID
  - `spec_no`: 商家编码
  - `goods_name`: 货品名称
  - `num`: 货品数量
  - `price`: 销售单价
  - `order_price`: 成交价
  - `discount`: 单件折扣金额
  - `total_amount`: 总价
  - `stock_reserved`: 库存保留情况（0未保留 3已保留待审核 4待发货 5预订单库存）
  - `is_gift`: 是否是赠品
  - `batch_no`: 批次号
  - `defect`: 是否残次品
  - `weight`: 预估单个货品重量
  - `volume`: 体积

#### **sales_trade_ex（销售订单扩展表）**
- **主键**: `rec_id` (bigint)
- **外键**: `trade_id` → `sales_trade.trade_id` (FK_sales_trade_ex_trade_id)
- **核心字段**:
  - `trade_id`: 关联订单主表
  - `track_logistics_no`: 追踪单号
  - `logistics_print_url`: 物流单对应URL
  - `invoice_print_url`: 发票对应URL

#### **sales_trade_order_ex（销售订单明细扩展表）**
- **主键**: `rec_id` (bigint)
- **外键**: `trade_order_id` → `sales_trade_order.rec_id`
- **核心字段**:
  - `trade_order_id`: 关联订单明细表
  - `product_label_pdf_url`: 平台货品标签
  - `box_pdf_url`: 平台箱唛

#### **sales_trade_log（订单操作日志表）**
- **主键**: `rec_id` (bigint)
- **外键**: `trade_id` → `sales_trade.trade_id`
- **核心字段**:
  - `trade_id`: 关联订单主表
  - `operator_id`: 操作员ID
  - `type`: 操作类型（5接口下单 7递交订单 15取消订单 20修改货品 25修改物流 30修改仓库 55审核订单等）
  - `data`: 原因列表的ID
  - `message`: 操作消息
  - `created`: 创建时间

#### **api_sales_trade（接口推送订单实体表）**
- **主键**: `rec_id` (bigint)
- **唯一键**: `UK_api_sales_trade` (`trade_no`, `owner_id`)
- **说明**: 用于接收外部系统（如ERP、OMS）推送的订单数据

#### **api_sales_order（接口推送订单明细表）**
- **主键**: `rec_id` (bigint)
- **说明**: 用于接收外部系统推送的订单明细数据

#### **stockout_order（出库单表）**
- **关联**: 通过 `src_order_type=1` 和 `src_order_id=trade_id` 关联销售订单
- **说明**: 销售订单递交仓库后生成出库单

### 1.2 表关系图

```
sales_trade (订单主表)
    ├── 1:N → sales_trade_order (订单明细表)
    │       └── 1:1 → sales_trade_order_ex (明细扩展表)
    ├── 1:1 → sales_trade_ex (订单扩展表)
    ├── 1:N → sales_trade_log (订单日志表)
    ├── 1:N → stockout_order (出库单表，通过src_order_id关联)
    └── N:1 → api_sales_trade (接口推送订单，通过trade_no关联)
            └── 1:N → api_sales_order (接口推送订单明细)
```

### 1.3 关键关联关系

1. **订单与明细**: `sales_trade.trade_id` = `sales_trade_order.trade_id`
2. **订单与出库单**: `sales_trade.trade_id` = `stockout_order.src_order_id` AND `stockout_order.src_order_type=1`
3. **订单与库存**: 通过 `sales_trade_order.spec_id` 和 `sales_trade.warehouse_id` 关联 `stock_spec` 表
4. **订单与接口数据**: `sales_trade.trade_no` = `api_sales_trade.trade_no`

## 2. 关键业务流程

### 2.1 订单创建流程

#### **流程1: 接口推送订单创建**
```
api_sales_trade/api_sales_order (接口推送)
    ↓
SP_OPENAPI_SALES_TRADE_INSERT (存储过程)
    ↓
sales_trade (创建订单主表)
    ↓
sales_trade_order (创建订单明细)
    ↓
I_RESERVE_STOCK (库存占用)
    ↓
sales_trade_log (记录日志)
```

**关键方法**:
- **存储过程**: `SP_OPENAPI_SALES_TRADE_INSERT`
- **库存占用**: `I_RESERVE_STOCK(P_TradeID, P_Type, P_WarehouseID, ...)`
  - P_Type: 2=未付款库存, 3=已保留待审核, 4=待发货, 5=预订单库存

#### **流程2: 手工建单**
```
SP_SALES_NEW_MANUAL (存储过程)
    ↓
api_trade/api_trade_order (创建原始单)
    ↓
I_DL_MAP_TRADE_GOODS (映射货品)
    ↓
sales_trade (创建订单)
    ↓
sales_trade_order (创建订单明细)
    ↓
I_RESERVE_STOCK (库存占用)
    ↓
sales_trade_log (记录日志)
```

**关键方法**:
- **存储过程**: `SP_SALES_NEW_MANUAL`
- **货品映射**: `I_DL_MAP_TRADE_GOODS`

### 2.2 订单审核流程

```
待审核订单 (trade_status=30)
    ↓
I_SALES_TRADE_CHECK (审核存储过程)
    ↓
检查库存充足性
    ↓
I_RESERVE_STOCK (更新库存占用状态: 3已保留待审核)
    ↓
UPDATE sales_trade SET trade_status=30
    ↓
UPDATE sales_trade_order SET stock_reserved=3
    ↓
sales_trade_log (记录审核日志, type=55)
```

**关键方法**:
- **存储过程**: `I_SALES_TRADE_CHECK(P_TradeID, P_bInterface, P_OperatorId, P_Type)`
- **批量审核**: `SP_SALES_TRADE_CHECK_OPER(P_TradeIds, P_CheckStep, P_Type)`
- **自动审核**: `SP_SALES_TRADE_AUTO_CHECK()`

### 2.3 订单递交仓库流程

```
已审核订单 (trade_status=30)
    ↓
I_SALES_TRADE_GENERATE_STOCKOUT (生成出库单)
    ↓
stockout_order (创建出库单)
    ↓
UPDATE sales_trade SET trade_status=55 (已确认)
    ↓
UPDATE sales_trade_order SET stock_reserved=4 (待发货)
    ↓
sales_trade_log (记录递交日志, type=7)
```

**关键方法**:
- **存储过程**: `I_SALES_TRADE_GENERATE_STOCKOUT(P_TradeID, P_Status)`
- **状态**: P_Status=55表示已确认

### 2.4 订单发货流程

```
待发货订单 (trade_status=85)
    ↓
I_STOCKOUT_CONSIGN_UNSALES (发货存储过程)
    ↓
UPDATE stockout_order SET status=95 (已发货)
    ↓
UPDATE sales_trade SET trade_status=95 (已发货)
    ↓
UPDATE sales_trade_order SET stock_reserved=0 (释放库存占用)
    ↓
库存扣减 (stock_spec)
    ↓
stock_change_history (记录库存变化)
    ↓
sales_trade_log (记录发货日志, type=115)
```

**关键方法**:
- **存储过程**: `I_STOCKOUT_CONSIGN_UNSALES(P_StockoutId)`
- **批量发货**: `SP_STOCKOUT_CONSIGN_UNSALES(P_StockoutIds)`

### 2.5 订单取消流程

```
任意状态订单
    ↓
sales_trade_cancel (取消操作)
    ↓
检查订单状态是否可取消
    ↓
I_RESERVE_STOCK (释放库存占用)
    ↓
UPDATE sales_trade SET trade_status=5, cancel_reason=...
    ↓
UPDATE sales_trade_order SET stock_reserved=0
    ↓
sales_trade_log (记录取消日志, type=15)
```

**关键方法**:
- **操作标识**: `SalesTradeModel.Cancel = "sales_trade_cancel"`

### 2.6 订单修改流程

#### **修改货品**
```
I_SALES_ORDER_UPDATE (修改订单明细)
    ↓
UPDATE sales_trade_order (更新货品信息)
    ↓
重新计算订单金额
    ↓
UPDATE sales_trade (更新订单总金额)
    ↓
sales_trade_log (记录修改日志, type=20)
```

**关键方法**:
- **存储过程**: `I_SALES_ORDER_UPDATE(P_TradeID, P_OrderID, P_OrderInfo)`

#### **修改物流**
```
SP_SALES_TRADE_CHG_LOGISTICS (修改物流)
    ↓
I_SALES_TRADE_CHG_LOGISTICS (内部修改物流)
    ↓
UPDATE sales_trade SET logistics_id=...
    ↓
sales_trade_log (记录修改日志, type=25)
```

**关键方法**:
- **存储过程**: `SP_SALES_TRADE_CHG_LOGISTICS(P_LogisticsID, P_TradeIDs, P_Force)`
- **操作标识**: `SalesTradeModel.ChangeLogistics = "sales_trade_chg_logistics"`

#### **修改仓库**
```
sales_trade_change_warehouse (切换虚拟仓)
    ↓
I_RESERVE_STOCK (释放旧仓库库存，占用新仓库库存)
    ↓
UPDATE sales_trade SET warehouse_id=...
    ↓
sales_trade_log (记录修改日志, type=30)
```

**关键方法**:
- **操作标识**: `SalesTradeModel.ChangeWarehouse = "sales_trade_change_warehouse"`

### 2.7 订单拆分流程

```
待拆分订单
    ↓
SP_SALES_TRADE_SPLIT (拆分订单)
    ↓
创建新订单 (sales_trade)
    ↓
分配订单明细 (sales_trade_order)
    ↓
重新计算订单金额
    ↓
I_RESERVE_STOCK (重新分配库存占用)
    ↓
sales_trade_log (记录拆分日志)
```

**关键方法**:
- **存储过程**: `SP_SALES_TRADE_SPLIT(P_TradeId, P_FlagId)`
- **自定义拆分**: `SP_SALES_TRADE_SPLIT_CUSTOM(P_SplitType, P_SplitMode, P_FlagId)`
- **操作标识**: `SalesTradeModel.Split = "sales_trade_split"`

### 2.8 订单合并流程

```
待合并订单列表
    ↓
SP_SALES_TRADE_MERGE (合并订单)
    ↓
合并订单明细 (sales_trade_order)
    ↓
重新计算订单金额
    ↓
I_RESERVE_STOCK (合并库存占用)
    ↓
删除被合并的订单 (或标记)
    ↓
sales_trade_log (记录合并日志)
```

**关键方法**:
- **存储过程**: `SP_SALES_TRADE_MERGE(P_TradeIds)`
- **操作标识**: `SalesTradeModel.Merge = "sales_trade_merge"`

### 2.9 订单冻结/解冻流程

```
订单冻结/解冻
    ↓
sales_trade_freeze (冻结操作)
    ↓
UPDATE sales_trade SET freeze_reason=...
    ↓
sales_trade_log (记录冻结日志)
```

**关键方法**:
- **操作标识**: `SalesTradeModel.FreezeOrUnFreeze = "sales_trade_freeze"`

## 3. 数据修改的业务操作

### 3.1 订单主表 (sales_trade) 修改操作

| 操作 | 方法/存储过程 | 修改字段 | 说明 |
|------|-------------|---------|------|
| **创建订单** | `SP_OPENAPI_SALES_TRADE_INSERT`<br>`SP_SALES_NEW_MANUAL` | 所有字段 | 创建新订单记录 |
| **审核订单** | `I_SALES_TRADE_CHECK` | `trade_status=30`<br>`checker_id` | 订单状态变更为待客审 |
| **递交仓库** | `I_SALES_TRADE_GENERATE_STOCKOUT` | `trade_status=55` | 订单状态变更为已确认 |
| **发货** | `I_STOCKOUT_CONSIGN_UNSALES` | `trade_status=95` | 订单状态变更为已发货 |
| **取消订单** | `sales_trade_cancel` | `trade_status=5`<br>`cancel_reason` | 订单状态变更为已取消 |
| **修改物流** | `SP_SALES_TRADE_CHG_LOGISTICS` | `logistics_id`<br>`logistics_code` | 修改物流公司 |
| **修改仓库** | `sales_trade_change_warehouse` | `warehouse_id` | 切换虚拟仓 |
| **修改标记** | `sales_trade_chg_flag` | `flag_id` | 修改订单背景色标记 |
| **修改信息** | `sales_trade_update` | `receiver_*`<br>`buyer_message`<br>`cs_remark`等 | 修改收件人、备注等信息 |
| **修改打印备注** | `sales_trade_update_print_remark` | `print_remark` | 修改打印备注 |
| **冻结/解冻** | `sales_trade_freeze` | `freeze_reason` | 冻结或解冻订单 |
| **拆分订单** | `SP_SALES_TRADE_SPLIT` | 创建新订单，原订单可能删除或修改 | 拆分订单 |
| **合并订单** | `SP_SALES_TRADE_MERGE` | 合并订单，被合并订单可能删除 | 合并订单 |
| **转预订单** | 相关存储过程 | `trade_status=25` | 转为预订单 |

### 3.2 订单明细表 (sales_trade_order) 修改操作

| 操作 | 方法/存储过程 | 修改字段 | 说明 |
|------|-------------|---------|------|
| **创建订单明细** | `SP_OPENAPI_SALES_TRADE_INSERT`<br>`I_SALES_ORDER_INSERT` | 所有字段 | 创建订单明细记录 |
| **修改订单明细** | `I_SALES_ORDER_UPDATE` | `spec_id`<br>`num`<br>`price`<br>`discount`<br>`actual_num`等 | 修改货品信息、数量、价格等 |
| **审核订单** | `I_SALES_TRADE_CHECK` | `stock_reserved=3` | 库存保留状态变更为已保留待审核 |
| **递交仓库** | `I_SALES_TRADE_GENERATE_STOCKOUT` | `stock_reserved=4` | 库存保留状态变更为待发货 |
| **发货** | `I_STOCKOUT_CONSIGN_UNSALES` | `stock_reserved=0` | 释放库存占用 |
| **取消订单** | `sales_trade_cancel` | `stock_reserved=0` | 释放库存占用 |
| **库存占用** | `I_RESERVE_STOCK` | `stock_reserved` (2/3/4/5) | 根据订单状态更新库存占用状态 |

### 3.3 库存相关修改操作

| 操作 | 方法/存储过程 | 修改表 | 说明 |
|------|-------------|-------|------|
| **占用库存** | `I_RESERVE_STOCK` | `stock_spec`<br>`sales_trade_order` | 更新 `unpay_num`/`order_num`/`sending_num`/`subscribe_num` |
| **释放库存** | `I_RESERVE_STOCK` | `stock_spec`<br>`sales_trade_order` | 减少库存占用数量 |
| **发货扣减库存** | `I_STOCKOUT_CONSIGN_UNSALES` | `stock_spec`<br>`stock_change_history` | 扣减实际库存，记录库存变化历史 |

### 3.4 日志表 (sales_trade_log) 修改操作

所有订单操作都会在 `sales_trade_log` 表中记录日志，包括：
- **type=1**: 客户下单
- **type=2**: 客户付款
- **type=5**: 接口下单
- **type=7**: 递交订单
- **type=15**: 取消订单
- **type=20**: 修改货品
- **type=25**: 修改物流
- **type=30**: 修改仓库
- **type=35**: 修改收件人信息
- **type=40**: 修改其他信息
- **type=55**: 审核订单
- **type=60**: 驳回审核
- **type=115**: 已发货
- **type=116**: 出库并发货

### 3.5 出库单表 (stockout_order) 修改操作

| 操作 | 方法/存储过程 | 修改字段 | 说明 |
|------|-------------|---------|------|
| **生成出库单** | `I_SALES_TRADE_GENERATE_STOCKOUT` | 创建出库单记录 | 订单递交仓库时生成 |
| **审核出库单** | `I_STOCKOUT_CHECK_UNSALES` | `status=50` | 出库单审核 |
| **发货** | `I_STOCKOUT_CONSIGN_UNSALES` | `status=95` | 出库单发货 |

## 5. 订单状态流转

```
10 等递交
    ↓
30 待客审 (审核)
    ↓
35 待财审 (可选)
    ↓
55 已确认 (递交仓库，生成出库单)
    ↓
65 已验货 (出库单验货)
    ↓
70 已打包 (出库单打包)
    ↓
75 已称重 (出库单称重)
    ↓
85 待发货 (出库单待发货)
    ↓
95 已发货 (发货完成)
    ↓
100 已签收
    ↓
110 已完成

特殊状态:
5  已取消
25 预订单
```

## 6. 库存占用状态 (stock_reserved)

- **0**: 未保留（取消的订单或完成）
- **2**: 未付款库存占用
- **3**: 已保留待审核
- **4**: 待发货
- **5**: 预订单库存

## 7. 数据流动总结

### 7.1 订单创建时的数据流
```
外部系统/手工输入
    ↓
api_sales_trade/api_sales_order (接口表)
    ↓
sales_trade/sales_trade_order (业务表)
    ↓
I_RESERVE_STOCK → stock_spec (库存占用)
    ↓
sales_trade_log (操作日志)
```

### 7.2 订单审核时的数据流
```
sales_trade (trade_status=30)
    ↓
I_SALES_TRADE_CHECK
    ↓
检查 stock_spec (库存充足性)
    ↓
I_RESERVE_STOCK → stock_spec (更新库存占用)
    ↓
sales_trade_order (stock_reserved=3)
    ↓
sales_trade_log (记录审核日志)
```

### 7.3 订单发货时的数据流
```
sales_trade (trade_status=85)
    ↓
stockout_order (出库单)
    ↓
I_STOCKOUT_CONSIGN_UNSALES
    ↓
stock_spec (扣减库存)
    ↓
stock_change_history (记录库存变化)
    ↓
sales_trade (trade_status=95)
    ↓
sales_trade_order (stock_reserved=0)
    ↓
sales_trade_log (记录发货日志)
```

## 8. 关键业务规则

1. **库存占用规则**: 订单创建后根据订单状态占用库存，取消或完成后释放
2. **状态流转规则**: 订单状态必须按顺序流转，不能跳跃
3. **审核规则**: 订单必须审核后才能递交仓库
4. **发货规则**: 订单必须生成出库单后才能发货
5. **拆分合并规则**: 拆分合并操作会重新分配库存占用
6. **日志记录规则**: 所有订单操作都会记录日志，便于追溯
