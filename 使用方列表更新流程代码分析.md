```csharp
/// <summary>
/// 更新使用方列表
/// </summary>
[Click("#btn_refresh_receiver_list")]
public void OnClickRefreshReceiverList()
{
    if (!RefreshReceiverList())
    {
        return;
    }

    OnClickSearch();
    Alert("更新成功！");
}
```

#### 1.3 核心更新方法

```csharp
/// <summary>
/// 更新使用方列表
/// </summary>
/// <param name="alterWhenError">更新出错时，是否弹窗报错</param>
/// <returns>更新成功</returns>
private bool RefreshReceiverList(bool alterWhenError = true)
{
    try
    {
        DB.Row("sys_ext_cmd", "share_logistics", "merchant_query", 0);
    }
    catch (Exception e)
    {
        if (alterWhenError)
        {
            Alert(e.Message);
        }

        return false;
    }

    return true;
}
```

**关键代码说明：**
- **调用方式：** `DB.Row("sys_ext_cmd", "share_logistics", "merchant_query", 0)`
  - `sys_ext_cmd` - 外部命令存储过程
  - `share_logistics` - 模块名称（共享物流模块）
  - `merchant_query` - 方法名称（查询商户/使用方列表）
  - `0` - 参数（0表示查询使用方，1表示查询提供方）

### 2. 调用机制

#### 2.1 `sys_ext_cmd` 存储过程

`sys_ext_cmd` 是一个外部命令调用机制，用于调用外部脚本或扩展代码。它通过以下方式工作：

1. **存储过程调用：** C# 代码调用 `DB.Row("sys_ext_cmd", "share_logistics", "merchant_query", 0)`
2. **模块路由：** 存储过程根据模块名 `share_logistics` 和方法名 `merchant_query` 路由到相应的实现
3. **外部脚本执行：** 可能通过以下方式实现：
   - **Squirrel 脚本：** 存储在 `sys_ext_resource` 表中（type=1）
   - **C# 扩展代码：** 存储在 `sys_ext_resource` 表中（type=4）
   - **外部接口调用：** 通过 HTTP 或其他方式调用平台接口

#### 2.2 数据同步流程

根据代码和表结构分析，同步流程应该是：

1. **调用平台接口：** `merchant_query` 方法调用共享物流平台的"获取提供方/使用方列表"接口
2. **获取数据：** 从平台接口获取最新的使用方列表数据（JSON格式）
3. **数据解析：** 解析返回的JSON数据
4. **数据库更新：** 将数据同步到 `share_logistics_receiver` 表
   - 如果 `merchant_id` 已存在，则更新记录
   - 如果 `merchant_id` 不存在，则插入新记录
   - 使用 `ON DUPLICATE KEY UPDATE` 或类似的逻辑处理

### 4. 数据表结构

**目标表：** `share_logistics_receiver`

**关键字段：**
- `merchant_id` - 使用方ID（唯一索引，由平台返回）
- `merchant_no` - 使用方编号
- `nickname` - 使用方名称
- `memo` - 使用方申请备注
- `our_remark` - 我方备注（WMS维护，不同步）
- `cooperate_state` - 合作状态（可能由平台同步，也可能由WMS审核操作修改）
- `employee_id` - 所属员工（WMS维护，不同步）
- `modified` - 修改时间（平台返回）
- `created` - 创建时间（平台返回）

由于 `sys_ext_cmd` 的具体实现代码不在当前代码库中（可能存储在数据库的 `sys_ext_resource` 表中，或作为外部扩展），同步逻辑应该是：

```sql
-- 伪代码，实际逻辑在外部脚本中
```

### 6. 调用时机

1. **窗口加载时：** 自动调用（静默模式，不显示错误）
2. **手动点击按钮：** 用户点击"更新使用方列表"按钮
3. **审核操作后：** 审核通过、拒绝合作、重新合作后会自动刷新列表

### 7. 错误处理

- **静默模式（alterWhenError=false）：** 窗口加载时调用，出错不弹窗，避免影响用户体验
- **交互模式（alterWhenError=true）：** 手动点击按钮时调用，出错会弹窗提示用户

### 8. 总结

更新使用方列表的核心代码流程：

1. **入口：** `ShareLogisticsWaybillReceiverWindow.RefreshReceiverList()`
2. **调用：** `DB.Row("sys_ext_cmd", "share_logistics", "merchant_query", 0)`
3. **执行：** 外部脚本/扩展代码调用平台接口获取数据
4. **同步：** 将数据同步到 `share_logistics_receiver` 表
5. **刷新：** 重新查询并显示最新数据

**关键点：**
- `sys_ext_cmd` 是外部命令调用机制，具体实现可能在数据库的 `sys_ext_resource` 表中
- 参数 `0` 表示查询使用方，参数 `1` 表示查询提供方
- 同步时会保留WMS内部维护的字段（`our_remark`、`employee_id`）
