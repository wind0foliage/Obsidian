# 使用方列表业务核心分析

## 1. 核心数据表及关系

### 1.1 主要数据表

#### `share_logistics_receiver` - 使用方列表（主表）
**表结构：**
- `receiver_id` - 主键，自增ID
- `merchant_id` - **使用方ID**（由平台返回，全局唯一，平台的数据库中对应的id）
- `merchant_no` - **使用方编号**（使用方商户编码）
- `nickname` - **使用方名称**
- `memo` - 使用方申请备注（使用方备注）
- `our_remark` - **我方备注**（WMS内部维护）
- `user_merchant_sid` - 使用方租户ID
- `user_merchant_pid` - 使用方租户的产品线编码
- `cooperate_state` - **合作状态**（0:待审核, 1:拒绝合作, 2:审核通过）
- `employee_id` - **所属员工**（默认是0表示全部）
- `modified` - 修改时间（平台返回）
- `created` - 创建时间（平台返回）
- `wms_created` - WMS创建时间
- `wms_modified` - WMS修改时间

**唯一索引：** `UK_share_logistics_receiver_merchant_id` (`merchant_id`)
**索引：** `IK_share_logistics_receiver_cooperate_state` (`cooperate_state`)

**数据来源：** 从"获取提供方/使用方列表"接口获取

#### `share_logistics_provider_logistics` - 提供方共享的物流（关联表）
**表结构：**
- `provider_logistics_id` - 主键，自增ID
- `cooperation_logistic_id` - **合作物流面单表主键id**（平台的数据库中对应的id，后续操作接口入参）
- `user_merchant_no` - **使用方编号**（关联 `share_logistics_receiver.merchant_no`）
- `shop_name` - 授权店铺名称
- `logistic_name` - 物流公司名称
- `branch_name` - 网点名称
- `province` - 网点所在省
- `city` - 网点所在市
- `district` - 网点所在区县
- `address_detail` - 网点地址
- `quantity` - **可用单量**
- `used_quantity` - **已用单量**
- `cancel_quantity` - **已回收单量**
- `status` - **状态**（0:启用, 1:未启用）
- `logistic_type` - 加盟类型（0:加盟, 1:直营）
- `memo` - 备注
- `auth_id` - 授权ID（后续操作接口用到）
- `waybill_source` - 单号来源（面单类型）
- `logistic_code` - 快递公司编码
- `branch_code` - 网点编码
- `wms_logistics_id` - **对应的WMS物流公司ID**（关联 `cfg_logistics.logistics_id`）
- `wms_bill_type` - 对应的WMS单号类型
- `wms_logistics_type` - 对应的WMS物流类型

**唯一索引：** `UK_share_logistics_provider_logistics_cooperation_logistic_id` (`cooperation_logistic_id`)
**索引：** `IK_share_logistics_provider_logistics_user_merchant_no` (`user_merchant_no`)

**数据来源：** 从"提供方查看使用方列表的合作物流信息"接口获取

#### `share_logistics_receiver_logistics_relation` - 使用方使用的共享物流和WMS系统物流的绑定关系

- `rec_id` - 主键，自增ID

**唯一索引：** `UK_share_logistics_relation_logistics_id` (`logistics_id`)
**索引：** `IK_share_logistics_relation_receiver_logistics_id` (`receiver_logistics_id`)

**数据来源：** 由WMS自行维护

#### `share_log` - 共享日志表

### 1.2 关联表

- 通过 `share_logistics_receiver.employee_id` 关联
- 存储员工信息

## 2. 关键业务流程

### 2.1 查询使用方列表流程

**入口：** `ShareLogisticsWaybillReceiverWindow` (使用方列表窗口)

**流程：**
1. **查询条件：**
   - 合作状态（下拉选择：全部、待审核、拒绝或停止合作、已合作）
   - 员工权限过滤：非管理员只能看到 `employee_id = 0` 或 `employee_id = 当前员工ID` 的记录

2. **查询存储过程：** `share_logistics_receiver_get`
   - 根据查询条件动态构建SQL
   - 支持分页查询
   - 返回字段：`receiver_id`, `merchant_id`, `merchant_no`, `user_merchant_sid`, `nickname`, `memo`, `our_remark`, `cooperate_state`, `modified`, `created`, `employee_id`

3. **数据展示：**
   - 主列表：显示使用方基本信息及合作状态
   - 共享物流标签页：显示选中使用方的共享物流信息（通过 `share_logistics_provider_logistics_query` 查询）
   - 日志标签页：显示操作日志（通过 `share_logistics_provider_get_log` 查询）

### 2.2 更新使用方列表流程

**入口：** 点击"更新使用方列表"按钮

**流程：**
1. **调用外部接口：** `sys_ext_cmd` → `share_logistics` → `merchant_query`
2. **同步数据：** 从平台接口获取最新的使用方列表数据
3. **更新数据库：** 更新或插入 `share_logistics_receiver` 表数据
4. **刷新界面：** 重新查询并显示最新数据

**关键代码：**
```csharp
DB.Row("sys_ext_cmd", "share_logistics", "merchant_query", 0);
```

### 2.3 审核流程

**审核通过：**
1. **前置条件：** 合作状态必须为 `0`（待审核）
2. **调用接口：** `sys_ext_cmd` → `share_logistics` → `change_cooperate_state`，参数：`merchantId`, `2`
3. **更新状态：** 将 `cooperate_state` 更新为 `2`（审核通过）
4. **刷新列表：** 重新加载使用方列表

**拒绝或停止合作：**
1. **前置条件：** 合作状态必须为 `0`（待审核）或 `2`（已合作）
2. **调用接口：** `sys_ext_cmd` → `share_logistics` → `change_cooperate_state`，参数：`merchantId`, `1`
3. **更新状态：** 将 `cooperate_state` 更新为 `1`（拒绝合作）
4. **刷新列表：** 重新加载使用方列表

**重新合作：**
1. **前置条件：** 合作状态必须为 `1`（拒绝合作）
2. **调用接口：** `sys_ext_cmd` → `share_logistics` → `change_cooperate_state`，参数：`merchantId`, `3`
3. **更新状态：** 将 `cooperate_state` 更新为 `0`（待审核）
4. **刷新列表：** 重新加载使用方列表

### 2.4 添加共享物流流程

**入口：** 点击"添加共享物流"按钮

**流程：**
1. **选择系统物流：** 从WMS系统物流列表中选择要共享的物流
2. **添加授权：** 调用接口 `authority_add` 添加授权
3. **刷新授权物流：** 调用接口 `authority_waybill_net_site_refresh` 刷新授权物流的网点信息
4. **查询合作物流：** 调用接口 `cooperation_logistics_list_query` 查询可用的合作物流
5. **选择授权物流：** 从授权物流列表中选择
6. **选择网点：** 选择网点名称和网点地址
7. **保存：** 调用接口 `cooperation_logistics_add` 添加共享物流
8. **刷新：** 刷新共享物流列表

**关键接口调用：**
- `authority_add` - 添加授权
- `authority_waybill_net_site_refresh` - 刷新网点信息
- `cooperation_logistics_list_query` - 查询合作物流列表
- `cooperation_branch_list_query` - 查询网点列表
- `cooperation_branch_address_list_query` - 查询网点地址列表
- `cooperation_logistics_add` - 添加共享物流

### 2.5 单量充值流程

**入口：** 在共享物流列表中选择物流，点击"单量充值"按钮

**流程：**
1. **打开充值窗口：** `ShareLogisticsChargeOrder`
2. **查询充值信息：** 调用存储过程 `share_logistics_charge_order_info` 获取充值相关信息
3. **查询可充值单量：** 调用接口 `cooperation_quantity_can_charge` 查询可充值单量（直营物流不需要查询）
4. **输入充值单量：** 用户输入充值单量和备注
5. **提交充值：** 调用接口 `branch_charge_add` 提交充值
6. **刷新列表：** 刷新共享物流列表

### 2.6 启用/停用共享物流流程

**入口：** 在共享物流列表中选择物流，点击"启用/停用"按钮

**流程：**
1. **调用接口：** `sys_ext_cmd` → `share_logistics` → `cooperation_logistic_status_update`，参数：`providerLogisticsId`
2. **更新状态：** 外部接口更新 `share_logistics_provider_logistics.status` 字段（0:启用, 1:未启用）
3. **刷新列表：** 刷新共享物流列表

### 2.7 刷新共享物流流程

**入口：** 点击"刷新共享物流"或"一键刷新共享物流"按钮

**流程：**
1. **刷新共享物流使用信息：** 调用接口 `refresh_share_logistics_usage_info` 刷新共享物流的使用情况
2. **刷新合作物流列表：** 调用接口 `cooperation_logistic_user_list` 刷新指定使用方的合作物流列表
3. **更新数据库：** 同步更新 `share_logistics_provider_logistics` 表数据
4. **刷新界面：** 重新显示最新数据

**业务操作：** 在使用方列表中添加或修改我方备注

**修改的表：**
- `share_logistics_receiver.our_remark` - 我方备注字段

**影响的业务：**
- 记录WMS内部对使用方的备注信息
- 记录操作日志到 `share_log` 表

**相关代码：**
- `ShareLogisticsWaybillReceiverWindow.OnClickAddOurRemark()` - 打开备注窗口
- `ShareLogisticsOurRemarkWindow.OnSave()` - 保存备注
- 存储过程：`share_logistics_receiver_add_our_remark`

**存储过程逻辑：**
```sql
UPDATE share_logistics_receiver 
SET our_remark = @tmp_our_remark 
WHERE merchant_id = @tmp_merchant_id;

INSERT INTO share_log (operator_id, merchant_id, message, user_type) 
VALUES (@cur_uid, @tmp_merchant_id, CONCAT('添加我方备注，', @tmp_our_remark), 1);
```

### 3.4 设置所属员工（批量修改）

**业务操作：** 批量设置使用方的所属员工

**修改的表：**
- `share_logistics_receiver.employee_id` - 所属员工字段

**影响的业务：**
- 控制员工权限：非管理员只能看到自己负责的使用方
- 记录操作日志到 `share_log` 表

**相关代码：**
- `ShareLogisticsWaybillReceiverWindow.OnSetEmployee()` - 设置所属员工
- 存储过程：`share_logistics_receiver_set_employee`

**存储过程逻辑：**
```sql
-- 记录日志
INSERT INTO share_log (operator_id, merchant_id, message, user_type) 
SELECT @cur_uid, slr.merchant_id, CONCAT('所属员工由【',IF(slr.employee_id = 0, '全部', he.shortname),'】修改为【',@tmp_employee_short_name,'】'), 1 
FROM share_logistics_receiver slr 
INNER JOIN tmp_xchg tx ON slr.receiver_id = tx.f1
LEFT JOIN hr_employee he ON slr.employee_id = he.employee_id 
WHERE slr.employee_id <> @tmp_employee_id;

-- 更新员工ID
UPDATE share_logistics_receiver slr 
INNER JOIN tmp_xchg tx ON slr.receiver_id = tx.f1 
SET slr.employee_id = @tmp_employee_id;
```

## 4. 关键业务规则

### 4.1 合作状态流转规则
- **待审核(0)** → **审核通过(2)**：通过审核操作
- **待审核(0)** → **拒绝合作(1)**：通过拒绝操作
- **已合作(2)** → **拒绝合作(1)**：通过停止合作操作
- **拒绝合作(1)** → **待审核(0)**：通过重新合作操作

### 4.2 员工权限规则
- 管理员（employee_id = 1）可以查看所有使用方
- 普通员工只能查看 `employee_id = 0`（全部）或 `employee_id = 当前员工ID` 的使用方
- 查询时自动过滤：`(slr.employee_id = 0 OR slr.employee_id = @cur_uid)`

### 4.3 单量充值规则
- 直营物流（logistic_type = 1）不需要查询可充值单量，可充值单量不受限
- 加盟物流（logistic_type = 0）需要查询可充值单量，充值单量不能超过可充值单量
- 充值单量不能为零

### 4.4 数据同步规则
- 使用方列表数据从平台接口同步，WMS不直接修改基础信息
- 共享物流数据从平台接口同步，单量、状态等信息由平台维护
- WMS可以修改的字段：`our_remark`（我方备注）、`employee_id`（所属员工）

### 4.5 日志记录规则
- 所有关键操作都会记录日志到 `share_log` 表
- 日志包含操作人、操作时间、操作内容等信息
- 支持查询历史操作记录

## 5. 相关权限

- `setting_share_logistics` - 共享面单设置（主权限）
- `setting_share_logistics_provider` - 我是提供方
- `share_logistics_provider_receiver_list` - 使用方列表
- `share_logistics_provider_receiver_search` - 搜索
- `share_logistics_cooperate_check` - 审核通过
- `share_logistics_cooperate_reject` - 拒绝或停止合作
- `share_logistics_cooperate_restart` - 重新合作
- `share_logistics_add_share_waybill` - 添加共享物流
- `share_logistics_disabled_change` - 启用/停用
- `share_logistics_charge_order` - 单量充值
- `share_logistics_update_authorizse` - 更新授权
- `share_logistics_update_authorizse_info` - 更新增值服务

## 6. 总结

使用方列表业务的核心是管理共享物流的使用方，主要涉及：
1. **数据存储：** 通过 `share_logistics_receiver` 表存储使用方信息，通过 `share_logistics_provider_logistics` 存储共享物流信息
2. **数据查询：** 支持按合作状态查询，支持员工权限过滤
3. **数据修改：** 主要通过外部接口同步数据，WMS可以修改备注和所属员工
4. **业务流程：** 包括审核、添加共享物流、单量充值、启用/停用等操作
5. **日志记录：** 所有关键操作都会记录日志，便于追溯

该业务与共享物流平台紧密集成，大部分数据通过外部接口同步，WMS主要负责展示和管理操作。
