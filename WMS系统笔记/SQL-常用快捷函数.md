
## 数据转换

精度转换-保留四位小数
CAST( 数值 AS DECIMAL(18,4))

小数转 - 整数
**字符串陷阱**
-- 错误示范：若 1.0000 是字符串 '1.0000' CAST('1.0000' AS INT) -- 部分数据库报错！ 

-- 正确做法： CAST(CAST('1.0000' AS DECIMAL(10,4)) AS INT) -- 先转数值再转整数

**NULL 处理**
CAST(NULL AS INT) 
-- 结果仍是 NULL 
-- 安全写法： 
COALESCE(CAST(num AS INT), 0)

| 环境         | 代码                                  | 说明                 |
| ---------- | ----------------------------------- | ------------------ |
| MySQL      | `CAST(1.0000 AS UNSIGNED)`          | 无符号整数（知识库[7]）      |
| SQL Server | `CONVERT(INT, 1.0000)`              | 与 CAST 等效（知识库[10]） |
| Python     | `int(1.0000)` 或 `round(1.0000)`     | 直接类型转换（知识库[5]）     |
| Excel      | `=INT(1.0000)` 或 `=ROUND(1.0000,0)` | 知识库[3]方法           |
| C语言        | `(int)1.0000f`                      | 强制类型转换（知识库[2]）     |


# `TRIM(TRAILING '.' FROM TRIM(TRAILING '0' FROM CAST(sod.num AS CHAR)))`

### `CAST(... AS CHAR)`

| 元素     | 作用                                 | 依据                                     |
| ------ | ---------------------------------- | -------------------------------------- |
| `CAST` | 显式类型转换函数（SQL-92标准）                 | 知识库[4][8]：等效于ODBC的`CONVERT`，用于安全转换数据类型 |
| `AS`   | 语法关键字，指定目标数据类型                     | SQL标准语法                                |
| `CHAR` | 目标数据类型（字符串）                        | 将数值转为字符串，使后续字符串操作生效                    |
| 效果     | `1.5000`（DECIMAL）→ `'1.5000'`（字符串） | 知识库[1][5][9]：数值转字符串是去零前提               |

### `TRIM(TRAILING '0' FROM ...)`
| 元素             | 作用                                                   | 依据                                  |
| -------------- | ---------------------------------------------------- | ----------------------------------- |
| `TRAILING '.'` | 删除尾部残留的小数点                                           | 知识库[1][5][9]：处理`'100.'` → `'100'`场景 |
| 效果             | `'100.'` → `'100'`  <br>`'1.5'` → `'1.5'`（尾部非`.`，不变） | 知识库[1]示例：`100.000000 → 100`         |
